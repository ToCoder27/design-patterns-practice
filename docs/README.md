## 学习建议

### 推荐顺序

```
第一阶段：单例 → 策略 → 工厂
    ↓
第二阶段：适配器 → 观察者 → 状态机 → 命令
    ↓
第三阶段：模板方法 → 装饰器 → 外观 → 建造者
```

### 每个模式的学习步骤

1. **阅读文档**：理解"一句话理解"和"要解决的问题"
2. **看代码**：阅读示例代码，理解核心实现
3. **编译运行**：进入对应目录编译并运行
4. **动手练习**：完成文档中的练习题

## 模式对比速查

### 策略模式 vs 状态模式

| 对比项 | 策略模式 | 状态模式 |
|--------|----------|----------|
| **目的** | 切换算法 | 状态转换 |
| **关系** | 客户端设置策略 | 状态自己决定转换 |

### 适配器模式 vs 外观模式

| 对比项 | 适配器模式 | 外观模式 |
|--------|----------|----------|
| **目的** | 接口转换 | 简化接口 |
| **对象** | 一个被适配者 | 多个子系统 |

### 观察者模式 vs 命令模式

| 对比项 | 观察者模式 | 命令模式 |
|--------|----------|----------|
| **目的** | 一对多通知 | 封装操作 |
| **特性** | 自动通知 | 支持撤销/宏命令 |

## 设计原则

| 原则 | 说明 | 体现模式 |
|------|------|----------|
| **开闭原则** | 对扩展开放，对修改关闭 | 策略、工厂、观察者 |
| **依赖倒置** | 依赖抽象，不依赖具体 | 工厂、命令、观察者 |
| **单一职责** | 一个类只负责一件事 | 命令、状态、装饰器 |
| **组合优于继承** | 优先使用组合而非继承 | 装饰器、策略、状态 |
